# retrospectiva

- Aplica el protocolo de `conocimientos/retrospectiva.md` solo después de que el usuario valide una obra.

## Cuándo se activa

- El usuario pide revisar qué se ha aprendido tras una obra aprobada.
- `proyecto` o `produccion` sugieren cerrar el ciclo tras aprobar.

## Sale

- Propuesta concreta: `Añadir`, `Modificar`, `Eliminar` o `No hacer nada`, esperando aprobación antes de tocar archivos.
